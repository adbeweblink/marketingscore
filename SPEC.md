## 1. 產品概述與目標用戶

### 產品定位

MarketingScore 是一個**通用型活動即時互動評分平台**，讓企業在聚餐、春酒、尾牙、Team Building 等場合，透過手機即時投票/評分，搭配大螢幕動態排行榜，創造沉浸式互動體驗。

### 一句話

> 「打開 LINE，掃碼入桌，投票評分，大螢幕即時跳分。」

### 目標用戶

| 角色 | 描述 | 痛點 |
|------|------|------|
| **活動主辦方** | 企業 HR、行銷部門、經銷商管理 | 傳統舉牌投票慢、不精確、沒數據留存 |
| **活動公司** | 企劃、執行團隊 | 每次活動重做系統、沒有可複用工具 |
| **主持人** | 控場者 | 需要即時看到結果、控制流程節奏 |
| **參與者** | 與會來賓 | 舉牌尷尬、想匿名投票、手機操作最自然 |

### 商業模式（初步）

| 模式 | 定價 | 適用 |
|------|------|------|
| 單場授權 | NT$3,000~8,000/場 | 中小企業自辦活動 |
| 活動公司方案 | NT$15,000~30,000/月 | 包月不限場次 |
| 客製方案 | 另議 | 品牌視覺客製、功能擴充 |

## 4. 遊戲模式詳細定義

### 模式架構

所有遊戲都是 `Round`（回合）的實例，每個 Round 綁定一個 `GameMode`。系統不硬編遊戲邏輯，而是透過模式組合實現彈性。

### 4.1 評分制（Rating Mode）

> 適用：團體自選曲、男女合唱 PK

| 參數 | 說明 | 預設值 |
|------|------|--------|
| `scaleMin` | 最低分 | 1 |
| `scaleMax` | 最高分 | 10 |
| `allowSelfVote` | 能否評自己桌/組 | false |
| `aggregation` | 計分方式 | `average`（平均）/ `sum`（加總）/ `trimmed_mean`（去頭尾平均） |
| `scoringUnit` | 評分對象 | `table`（桌）/ `group`（組）/ `individual`（個人） |

**流程**：
1. 主持人按「開始評分」
2. 參與者手機出現滑桿（或星星），選分數送出
3. 大螢幕即時顯示已投票人數（不顯示分數）
4. 主持人按「揭曉」→ 大螢幕動畫揭曉排名
5. 積分寫入對應桌/組

### 4.2 猜謎投票制（Guess Mode）

> 適用：蒙面歌手

| 參數 | 說明 | 預設值 |
|------|------|--------|
| `question` | 題目描述 | 「猜猜這是哪一桌？」 |
| `options` | 選項列表 | 自動生成（桌號列表） |
| `correctAnswer` | 正確答案 | 主持人設定 |
| `pointsCorrect` | 猜對得分 | 10 |
| `pointsPerformer` | 表演者被猜中得分 | 5 |
| `allowSelfGuess` | 能否猜自己桌 | false |

**流程**：
1. 表演者（匿名）表演完畢
2. 主持人按「開始猜測」
3. 參與者手機出現桌號選項，選一個送出
4. 主持人按「揭曉」→ 大螢幕公布正確答案 + 各桌猜對人數
5. 猜對者的桌加分，表演者的桌也加分

### 4.3 主持人裁決制（Host Decision Mode）

> 適用：飆高音挑戰、歡呼聲決勝

| 參數 | 說明 | 預設值 |
|------|------|--------|
| `candidates` | 候選名單 | 桌號或個人 |
| `scoreInput` | 裁決方式 | `rank`（排名）/ `points`（直接給分） |

**流程**：
1. 表演/比賽進行
2. 主持人在後台直接輸入排名或分數
3. 大螢幕顯示主持人裁決結果動畫
4. 積分寫入

### 4.4 自訂積分制（Custom Mode）

> 適用：任何非標準計分情境

| 參數 | 說明 |
|------|------|
| `customFields` | 自訂欄位（如加分事由、分數） |
| `formula` | 計分公式（支援四則運算） |

主辦方可在後台定義任意計分邏輯，系統不限制。

## 5. 資料模型草案

```
Event（活動）
├── id: UUID
├── name: string              // "Adobe FY26 經銷商大會春酒"
├── date: datetime
├── status: enum              // draft | active | completed
├── settings: JSON            // 品牌客製、防作弊規則
│
├── Table[]（桌次）
│   ├── id: UUID
│   ├── number: int           // 桌號 1~8
│   ├── name: string          // 可選別名 "戰鬥桌"
│   └── Participant[]
│       ├── id: UUID
│       ├── lineUserId: string
│       ├── displayName: string
│       └── joinedAt: datetime
│
├── Group[]（分組，多桌組隊用）
│   ├── id: UUID
│   ├── name: string          // "A 組"
│   └── tableIds: UUID[]      // [table1, table2]
│
├── Round[]（回合）
│   ├── id: UUID
│   ├── order: int            // 回合順序
│   ├── name: string          // "蒙面歌手（女生組）"
│   ├── gameMode: enum        // rating | guess | host_decision | custom
│   ├── config: JSON          // 該模式的參數
│   ├── status: enum          // pending | voting | reveal | completed
│   ├── scoringUnit: enum     // table | group | individual
│   ├── timerSeconds: int?    // 倒數秒數（null = 手動結束）
│   │
│   ├── Performance[]（本輪表演/參賽者）
│   │   ├── id: UUID
│   │   ├── performerType: enum   // table | group | individual
│   │   ├── performerId: UUID     // 對應 table/group/participant ID
│   │   ├── label: string         // 顯示名稱
│   │   └── finalScore: float?    // 最終計算分數
│   │
│   └── Vote[]（投票/評分）
│       ├── id: UUID
│       ├── participantId: UUID   // 投票者
│       ├── performanceId: UUID   // 投給誰
│       ├── value: JSON           // { score: 8 } 或 { guess: "table3" }
│       ├── createdAt: datetime
│       └── isValid: boolean      // 防作弊驗證結果
│
└── ScoreBoard（積分板）
    ├── entityType: enum      // table | group
    ├── entityId: UUID
    ├── totalScore: float
    └── roundScores: JSON     // { round1: 10, round2: 15, ... }
```

### 關鍵約束

- **一人一票**：同一 `participantId` + `roundId` 只能有一筆 `Vote`
- **不能投自己**：`Vote.participantId` 所屬桌/組 ≠ `Vote.performanceId` 所屬桌/組
- **投票時段鎖定**：`Round.status !== 'voting'` 時拒絕寫入

## 6. 頁面清單

### 6.1 Admin 後台（Web，桌機/平板）

| 頁面 | 路徑 | 說明 |
|------|------|------|

### 6.2 參與者手機頁（LIFF / LINE 內建瀏覽器）

| 頁面 | 路徑 | 說明 |
|------|------|------|
| 入場頁 | `/join/:eventCode` | 掃 QR 或輸入代碼，LINE 登入，選桌次 |
| 等待頁 | `/play/waiting` | 「請等待主持人開始下一輪」+目前排行榜 |
| 投票頁 | `/play/vote` | 依遊戲模式顯示：滑桿評分 / 選項猜測 |
| 結果頁 | `/play/result` | 本輪結果 + 你的投票 vs 正確答案 |
| 排行榜 | `/play/leaderboard` | 即時總排名 |

### 6.3 大螢幕投影頁（全螢幕，16:9）

| 頁面 | 路徑 | 說明 |
|------|------|------|
| 待機畫面 | `/display/:eventCode/idle` | 品牌 Logo + 活動名稱 + QR Code（供掃碼加入） |
| 回合標題 | `/display/:eventCode/round-intro` | 本輪名稱 + 遊戲說明動畫 |
| 投票進行中 | `/display/:eventCode/voting` | 即時投票進度條（已投 X/Y 人）+ 倒數計時 |
| 結果揭曉 | `/display/:eventCode/reveal` | **重頭戲** — 分數從低到高揭曉、金色粒子特效、排名動畫 |
| 總排行榜 | `/display/:eventCode/final` | 冠亞季軍動畫、煙火特效、各桌總分 |

### 大螢幕視覺規格

- **風格**：金碧輝煌、深色底 + 金色/琥珀色文字、粒子動畫
- **字體大小**：桌號/分數至少 72px，遠距離可讀
- **動畫**：分數跳動（counter animation）、排名滑動、揭曉翻牌、煙火/金色粒子
- **音效**：倒數滴答、揭曉鼓聲、冠軍號角（可選，由主辦方控制）

## 7. LINE Bot 互動流程

### 7.1 加入活動

```
用戶掃 QR Code
  → 開啟 LIFF 頁面（/join/:eventCode）
  → 自動取得 LINE userId + displayName
  → 選擇桌次（下拉選單）
  → 送出 → 綁定完成
  → LINE Bot 發送歡迎訊息：
    「✅ 已加入【Adobe FY26 春酒】第 3 桌！
     活動開始時會通知你～」
```

### 7.2 投票通知（P1，MVP 可先用 LIFF 輪詢）

```
主持人按「開始投票」
  → LINE Bot 推播給所有參與者：
    「🎤 第 2 輪【蒙面歌手（女生組）】開始投票！
     👉 點這裡投票 [LIFF 連結]」
  → 用戶點連結 → 開啟投票頁
```

### 7.3 結果通知（P1）

```
主持人按「揭曉結果」
  → LINE Bot 推播：
    「🏆 第 2 輪結果揭曉！
     🥇 第 5 桌 — 8.7 分
     🥈 第 3 桌 — 8.2 分
     你們桌排第 4 名（7.1 分）
     目前總積分：第 3 桌 42 分（第 2 名）」
```

### 7.4 技術方案

| 元件 | 選擇 | 原因 |
|------|------|------|
| LINE Login | LIFF v2 | 免輸入帳密、自動取 userId |
| 投票互動 | LIFF 內嵌 Web App | 比 Flex Message 靈活、UI 可控 |
| 通知推播 | LINE Messaging API（Push） | 主動推送、免費額度 500 則/月（超過需付費） |
| Rich Menu | 活動期間顯示 | 「投票」「排行榜」「我的桌次」 |

### LINE 帳號需求

- LINE Official Account（免費版即可，推播有月額度限制）
- LIFF App 註冊（LINE Developers Console）
- Channel Access Token + Channel Secret

## 8. 非功能需求

### 效能

| 指標 | 目標 | 說明 |
|------|------|------|
| 同時在線 | 100+ 人 | 8 桌 × 12 人 = 96，抓 120 |
| 投票到大螢幕更新 | < 1 秒 | WebSocket 即時推送 |
| 投票頁載入 | < 2 秒 | 手機 4G 環境 |
| 大螢幕動畫 | 60fps | CSS/Canvas 動畫，不依賴影片檔 |

### 可靠性

| 指標 | 目標 |
|------|------|
| 活動期間可用性 | 99.9%（活動通常 2-3 小時） |
| 投票不丟失 | 寫入確認機制，失敗重試 |
| 斷線重連 | WebSocket 自動重連 + 狀態恢復 |
| 離線容錯 | 手機短暫斷網 → 本地暫存 → 恢復後補送 |

### 安全

| 項目 | 做法 |
|------|------|
| 身份驗證 | LINE Login（OAuth 2.0） |
| 防重複投票 | Server 端 unique constraint（userId + roundId） |
| 防篡改 | 投票後不可修改 |
| 資料隔離 | 不同活動資料完全隔離 |
| HTTPS | 強制（LINE LIFF 要求） |

### 相容性

| 環境 | 要求 |
|------|------|
| 手機瀏覽器 | LINE 內建瀏覽器（iOS/Android）為主 |
| 大螢幕 | Chrome/Edge 最新版，1920×1080 |
| Admin 後台 | Chrome/Edge/Safari 最新版 |

## 9. 技術限制與風險

### 已知限制

| 限制 | 影響 | 緩解 |
|------|------|------|
| LINE 推播免費額度 500 則/月 | 多輪 × 100 人可能超額 | MVP 先用 LIFF 輪詢，不依賴推播；或升級 LINE OA 方案 |
| LIFF 只能在 LINE App 內開啟 | 非 LINE 用戶無法參與 | 提供備用 Web 入口（輸入暱稱 + 桌號） |
| 場地 Wi-Fi 不穩定 | 投票延遲或失敗 | 本地暫存 + 重試；建議主辦方準備 4G 分享器 |
| 大螢幕解析度不一 | 版面跑掉 | 固定 16:9 比例，CSS 等比縮放 |

### 風險評估

| 風險 | 機率 | 衝擊 | 對策 |
|------|------|------|------|
| 活動當天伺服器掛掉 | 低 | 極高 | 部署到可靠雲端（Vercel/Railway）+ 備用靜態頁 |
| LINE API 異常 | 低 | 高 | 備用 Web 入口不依賴 LINE |
| 參與者不會操作 | 中 | 中 | 入場時發操作說明卡、QR Code 直達 |
| 同時大量投票湧入 | 中 | 中 | WebSocket + 資料庫批次寫入、樂觀鎖 |
| 主持人操作失誤 | 中 | 中 | 「上一步」回退功能、操作確認彈窗 |

## 附錄 B：競品參考

| 產品 | 特色 | 我們的差異 |
|------|------|-----------|
| Slido | 投票/Q&A | 無遊戲模式、無桌次概念 |
| Kahoot! | 搶答遊戲 | 偏教育、無評分制、無 LINE 整合 |
| Mentimeter | 即時投票 | 無遊戲引擎、無大螢幕動畫 |
| AhaSlides | 互動簡報 | 功能分散、無台灣在地整合 |

**MarketingScore 的獨特價值**：
1. **遊戲引擎**：不只是投票，是完整的遊戲流程控制
2. **LINE 原生整合**：台灣人人有 LINE，零門檻
3. **大螢幕體驗**：金碧輝煌的動畫，不是簡陋的長條圖
4. **桌次邏輯**：專為聚餐場景設計，支援分組、防作弊
